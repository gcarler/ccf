# SKILL: Frontend CCF — Reglas para Agentes de IA

**Nombre:** ccf-frontend
**Descripción:** Skill con las reglas de frontend de la plataforma CCF. Úsala cuando trabajes en componentes, páginas o estilos del frontend de CCF (Next.js + Tailwind CSS). También aplícala al crear o modificar rutas, componentes de UI, y cualquier interacción con el sistema de diseño de CCF.

---

## Reglas Inquebrantables

Estas reglas son vinculantes para cualquier agente que toque archivos del frontend de CCF:

### 1. Personas ≠ Miembros
- En la UI **nunca** uses "miembro" para referirte a una persona. Usa "persona", "integrante" o "participante".
- No hay componente `MemberCard`, solo `PersonaCard`.
- No existe `members_count` en UI, solo `total_personas`.

### 2. Grupos de Evangelismo
- En UI pública: "Casas de Gloria" o "Grupos de Evangelismo"
- En UI de plataforma: "Grupos de Evangelismo"
- En API: endpoints `/community/grupos` y `/evangelism/grupos`

### 3. Drawers, no Modales
- **Prohibido:** `<Dialog>`, `<Modal>` para crear, editar o ver detalles.
- **Obligatorio:** `<Drawer>`, `<SidePanel>`, `<RightPanel>` para paneles laterales.
- Para contenido anidado: "Drawer Stacking" (apilar drawers).

### 4. Tokens Semánticos, no Colores Fijos
- **Prohibido:** `bg-blue-500`, `border-gray-200`, `text-white`, etc.
- **Obligatorio:** `bg-[hsl(var(--surface-1))]`, `border-[hsl(var(--border))]`, `text-[hsl(var(--text-primary))]`.

### 5. Carga Perezosa por Pestañas
- Perfiles de persona: dividir en tabs (CRM, Academia, Proyectos, Agenda).
- Cada tab carga su propia llamada API.
- **Prohibido:** un solo endpoint `/full-profile`.

### 6. Cero Modales de Creación
- **Prohibido:** `UniversalCreationModal.tsx` y cualquier modal para crear recursos.
- Usar Drawer lateral o página dedicada.

### 7. Convención de Rutas
- Formato: `/plataforma/{modulo}[/{submodulo}][/{id}]`
- Ejemplos: `/plataforma/personas`, `/plataforma/community/grupos`, `/plataforma/crm`

### 8. Llamadas API
- Usar siempre `apiFetch('/endpoint')` de `@/lib/http`.
- No usar `fetch()` directo.

### 9. Design System
- Usar componentes de `frontend/src/design/components/`: `DSButton`, `DSBadge`, `DSCard`, `DSMetric`, etc.

### 10. Arquitectura Multi-Tenant
- Toda vista de datos sensibles debe considerar el `sede_id` del usuario autenticado.
- No mostrar datos de sedes cruzadas.
